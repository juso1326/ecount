# 修改目標 0302v1 — 全站測試修改記錄

## 一、已修復問題

### 1. ✅ 應付帳款狀態篩選 Bug（`pending` vs `unpaid`）
- **問題**：`PayableController` 使用 `whereIn('status', ['pending', 'partial'])` 過濾「待付」，但資料庫實際值為 `unpaid`，導致逾期/快速篩選無效
- **修復**：`PayableController` 全部改為 `['unpaid', 'partial']`
- **修復**：應付帳款 index 頁面下拉選項 `value="pending"` → `value="unpaid"`

### 2. ✅ 應收帳款狀態資料不一致（4筆 `pending` → `unpaid`）
- **問題**：Seeder 早期產生 4 筆 `status='pending'`，但系統使用 `unpaid`
- **修復**：直接 UPDATE 資料庫；index 頁面下拉改為 `value="unpaid"`

### 3. ✅ 應付帳款 `payee_type` 新舊不一致（`company` vs `vendor`）
- **問題**：75 筆舊資料用 `payee_type='company'`，新表單產生 `vendor`；`FinancialReportController` 和 `DashboardController` 僅查 `company`，造成新資料被漏算
- **修復**：FinancialReport 和 Dashboard 改用 `whereIn(['company','vendor'])`
- **修復**：`Payable::getPayeeNameAttribute()` 同時處理 `member`/`vendor`/`expense` 類型

### 4. ✅ 公司編輯頁面缺少 Logo 上傳 UI
- **問題**：`companies/edit.blade.php` 沒有 logo 上傳區塊，但 Controller 已有處理邏輯
- **修復**：加入帶虛線框樣式的上傳區、現有圖片預覽、刪除按鈕

### 5. ✅ 應付帳款表單完整重寫（先前版本）
- **問題**：表單樣式混亂、JS 有 `#manager_name` 非法引用、重複事件綁定
- **修復**：全面重寫為 flat grid 佈局；修正 JS Bug

### 6. ✅ 專案預算小數點支援
- **問題**：`Project` 模型將 `budget` cast 為 `integer`，但 DB column 為 `decimal(15,2)`
- **修復**：cast 改為 `decimal:2`

## 二、待觀察 / 潛在問題

### A. 6 筆空白應付帳款（ID 1-6）
- `payee_type` 為空、無付款代碼、無對象，為早期測試資料
- **建議**：若影響報表，可考慮刪除或補充資料

### B. 應付帳款 `payee_type='company'` 共 75 筆
- 新表單已改用 `vendor`，但舊資料保留 `company`
- Index 頁面、FinancialReport 均已處理 `company`+`vendor` 組合
- **建議**：如需統一，可執行 `UPDATE payables SET payee_type='vendor' WHERE payee_type='company'`（需同步更新 FinancialReport 移除 company 判斷）

### C. 應收帳款 overdue 標記
- 系統目前無自動將逾期 `unpaid` 更新為 `overdue` 的排程
- 查詢時透過 `due_date < now()` 動態篩選，顯示層不會自動變色
- **建議**：考慮加入定時任務更新 `status='overdue'`
